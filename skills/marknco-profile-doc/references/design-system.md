# 마크앤컴퍼니 기업 프로필 디자인 시스템

> 이 문서를 Claude에게 주면 다른 엑셀/다른 목차에도 동일한 디자인으로 문서를 생성할 수 있습니다.
> 현재 적용 프로젝트: **2026 KT K-PATH 롱리스트** (v6 기준, 2026-07-03 확정)

## 1. 기본 설정

| 항목 | 값 |
|------|-----|
| 라이브러리 | `docx` npm 패키지 (v9.x) |
| 변환 방식 | `.docx` 생성 후 LibreOffice(`soffice`)로 PDF 변환 |
| 폰트 | **Pretendard** (OTF 파일 필요 — Regular / Bold 최소 2종) |

| 여백(MARGIN) | 1080 DXA (사방 동일, ≒ 1.9cm) |
| 목표 페이지 수 | 기본 **1페이지**. 고용·재무·매출 등 "성장 데이터" 섹션을 붙이면 **2페이지**(자동 페이지브레이크) |
| 파일명 규칙 | `[프로그램명]_[문서종류]_[기업명].docx/pdf` |

## 3. 타이포그래피

모든 텍스트는 `font: "Pretendard"`.  
docx의 size 단위는 **반포인트(half-point)** → `size: 21` = 10.5pt.

| 역할 | size 값 | 실제 pt | 스타일 | 색상 |
|------|---------|---------|--------|------|
| 기업명 (타이틀) | 38 | 19pt | Bold | `111315` |
| 서비스명 (부제) | 23 | 11.5pt | Bold | `ACCENT` |
| 섹션 헤딩 | 24 | 12pt | Bold | `ACCENT` |
| 표 라벨 | 20 | 10pt | Bold | `ACCENT` |
| 표 값 | 20 | 10pt | Regular | `BODY` |
| 본문 | 21 | 10.5pt | Regular | `BODY` |
| 불릿 항목 | 21 | 10.5pt | Regular | `BODY` |
| 외부 패널 레이블 (레거시) | 18 | 9pt | Bold | `PANEL_ACCENT` |
| 외부 패널 본문 (레거시) | 19 | 9.5pt | Regular | `3A3A3A` |
| 헤더·푸터 | 15–16 | 7.5–8pt | Regular | `SUBTLE` |

> **빈 값 표시**: 모든 필드 공통으로 `"-"` (SUBTLE 색). 예전엔 "정보 없음"/"해당 없음" 문구를 썼으나 v5부터 `"-"`로 통일.
> **외부 패널(레거시)**: 베이지 배경 박스 + 이탤릭 스타일은 초기 버전 디자인. v5부터는 패널 대신 `sections`/`linkSection`으로 같은 정보를 다른 섹션과 동일한 톤(헤딩+본문, 이탤릭 없음)으로 보여주는 것을 기본값으로 권장한다. 패널은 하위호환을 위해 남겨둠(`panel.enabled: true`로 여전히 사용 가능).

## 5. 컴포넌트별 규칙

### 5-1. 헤더 / 푸터
- **헤더**: 좌측 `프로그램명`, 우측 탭으로 밀어서 `날짜 작성` / 하단에 LINE 색 얇은 선
- **푸터**: 가운데 정렬, `출처 안내 문구 — {페이지번호}` 형식
- 날짜는 스크립트 실행 시점 자동 삽입 (`YYYY.MM.DD` 형식)
- 헤더 텍스트 바꾸는 법: `buildDoc()` 내 `TextRun({ text: "여기 수정" })`

### 5-2. 타이틀 블록
- 기업명 → 서비스명 순서, 사이에 여백 없음
- 서비스명 단락에 `border.bottom` (size: 10, ACCENT 색) 으로 구분선 표현
- **"eyebrow 라벨"(예: "2026 K-PATH 지원기업 프로필") 은 사용하지 않음** (제거됨, v4 기준)

### 5-3. 오버뷰 표 (개요 정보 테이블)
```
컬럼 너비: 라벨 2400 DXA / 값 (CONTENT_WIDTH - 2400) DXA
라벨 셀: ACCENT_LIGHT 배경, Bold, ACCENT 색, 상하여백 90
값 셀:   흰 배경, Regular, BODY 색, 상하여백 90
테두리:  LINE 색, 2pt 단선
링크:    ExternalHyperlink + LINK 색 + underline
빈값:    "-" (SUBTLE 색)
```
- 항목 수·라벨은 프로젝트마다 자유롭게 변경
- 금액 필드는 `toLocaleString("ko-KR") + "원"` 으로 포맷

#### 5-3-1. 항목이 많을 때 — 2단 나란히 배치 (`overviewGroups`)
항목이 10개 안팎을 넘으면 표 하나(세로로 긴 단일 컬럼)보다 좌/우 두 개의 작은 표를 나란히 배치하는 편이 공간 효율이 훨씬 좋다.
```
바깥 wrapper: 3컬럼 Table (테두리 전체 NONE) — [좌측 표 셀] [GAP 360 DXA] [우측 표 셀]
각 셀 안에 중첩 Table(nested Table)을 하나씩 배치 — docx는 TableCell 안에 Table 중첩 가능
그룹당 라벨 컬럼: 2100 DXA (고정), 값 컬럼: groupW - 2100
```
- **행 하나를 크게**: 특정 행에 `"span": 2`를 주면 그 행만 2배 높이(margin 자동 계산: `210×n - 120`)로 표시된다. 좌우 표의 "항목 개수" 자체가 구조적으로 다를 때(6개 vs 7개처럼) 쓰는 방법.
- **값 길이 편차가 큰 필드는 표 밖으로**: 성과키워드처럼 회사마다 글자 수 편차가 아주 큰 필드를 좌우 표 안에 넣으면, 그 필드가 긴 회사만 줄바꿈되어 표 높이가 어긋난다. 이런 필드는 `wideOverviewRows`로 표 아래 전체 폭 한 줄로 분리한다.
- **좌우 표 높이를 실측 기반으로 맞추기**: 위 두 방법으로도 안 맞거나(또는 표 구성을 회사마다 다르게 하고 싶지 않을 때), `scripts/balance_overview_groups.py`가 렌더링 결과를 실제로 측정해서 자동 보정한다.
  - 글자 수·폭 계산으로 줄바꿈을 예측하는 방식은 **폰트가 프로포셔널이라 부정확함이 실제로 확인됨**(같은 26자 URL도 문자열에 따라 줄바꿈 여부가 갈림). 반드시 실측할 것.
  - 보정값은 `company.__overviewFillerTwips = { "0"|"1": twips }` 형태로 companies.json에 저장되고, 스크립트가 이를 읽어 **부족한 쪽 그룹의 모든 행에 균등하게(반올림 오차만 마지막 행이 흡수)** 마진을 늘려 채운다.
  - 채택하지 않은 방식(시행착오 기록, 되돌리지 말 것): ① 테두리를 투명하게 처리한 빈 행 추가 — 공간은 차지하지만 눈에는 표가 짧아 보임. ② 빈 행에 보이는 테두리를 그리는 방식 — "빈 칸처럼 보인다"는 사용자 피드백으로 폐기. ③ 부족분을 마지막 행 하나에 몰아주는 방식 — 그 행만 유난히 커 보여서 "균등하게" 요청받음. **현재 방식(모든 행에 균등 분배)이 최종 확정.**

### 5-4. 섹션 헤딩
```
before: 240, after: 100
border.bottom: SINGLE, size 6, ACCENT 색, space 4
텍스트: Bold, 24(12pt), ACCENT 색
```
- 헤딩 아래에 본문 단락 또는 불릿 리스트 배치
- 원래 `before: 300, after: 120`이었으나, 1페이지 안에 안 들어가는 회사가 생겨(마지막 줄이 2페이지로 넘침) v5에서 축소. 그래도 넘치면 이 값을 더 줄이거나 5-5의 마지막 줄 여백을 줄인다.

### 5-5. 본문 단락 (`bodyPara`)
- 줄바꿈(`\n`)을 단락 분리로 처리
- 마지막 줄 `after: 100`, 중간 줄 `after: 40`, `line: 300` (v5에서 130→100으로 축소, 이유는 5-4와 동일)

### 5-6. 불릿 리스트 (`bulletList`)
```
불릿 문자: "•"
들여쓰기: left 320, hanging 220 DXA
항목 간격: after 60
빈 배열 시: "-" (SUBTLE 색) 출력
```

### 5-7. 2컬럼 나란히 배치 (관심 트랙 / 기대사항 패턴)
```
3컬럼 Table (테두리 전체 NONE) 활용
[왼쪽 내용] [400 DXA 간격 셀] [오른쪽 내용]
각 절반 너비: (CONTENT_WIDTH - 400) / 2
```
- 좌우 어떤 블록이든 이 패턴으로 나란히 배치 가능
- 오버뷰 표를 2단으로 나란히 배치할 때(5-3-1)도 같은 원리(3컬럼 wrapper table)를 쓰되, 셀 안에 불릿 리스트 대신 중첩 Table을 넣는다는 점만 다르다.

### 5-8. 외부 데이터 패널 (레거시, 선택적)
- **데이터가 있을 때만** 렌더링 (없으면 빈 배열 반환 → 아무것도 안 나옴)
- 위치: 오버뷰 표 바로 아래
- 배경: PANEL_FILL (`F6F4ED`), 테두리: `E3DCC6` (웜 베이지)
- 표시 항목 예시: `AI 요약` / `주요 투자자` / `혁신의숲 링크`
- 항목 라벨: Bold, PANEL_ACCENT 색 / 값: Regular, `3A3A3A` (v4까지는 Italic이었으나 "다른 섹션과 톤이 다르다"는 피드백으로 v5에서 제거)
- **v5 권장 대안**: 새 프로젝트는 패널 대신 `sections`(각 항목을 일반 섹션 헤딩+본문으로) + `linkSection`(링크 하나만 있는 항목)으로 구성해서, 문서 전체가 동일한 헤딩 스타일로 통일되도록 한다. 실제로 K-PATH 롱리스트 프로젝트에서 패널 박스를 쓰다가 "페이지가 어색하다"는 피드백을 받고 이 방식으로 전환했다.

### 5-9. 헤더 / 푸터 켜고 끄기
- `config.header.enabled`, `config.footer.enabled`를 `false`로 주면 아예 렌더링하지 않는다 (기본값은 `true`).
- 헤더/푸터를 끄면 본문이 차지할 수 있는 세로 공간이 그만큼 늘어나므로, 1페이지에 빠듯하게 들어가는 프로젝트에서 유용하다.

### 5-10. 성장 데이터 섹션 (`dataTables` — 고용·재무·매출, v6 신규)
```
위치: 기본 프로필(1페이지) 뒤에 자동 페이지브레이크 → 2페이지째
구성: [섹션 헤딩] → [기업별 차트 이미지(선택)] → [연도/기간 × 지표 매트릭스 표(선택)]
```
- **차트**: matplotlib으로 `scripts/generate_growth_charts.py`가 기업별 PNG를 미리 생성 → `chartField`로 회사마다 다른 이미지를 동적 매칭 (`chart.path` 정적 방식은 여러 회사 배치에 쓰면 안 됨 — 4-2 참고).
  - 색상: 고용(입사자 `#10B981` 민트 / 퇴사자 `#D7DCE3` 연회색 / 근무인원수 선 `#6C7FE0` 블루), 재무(자산 NAVY / 부채 CORAL / 자본 TEAL), 매출·손익(매출액 `#1ABC9C` / 영업이익 `#7B8FE8` / 순이익 `#F2C230`)
  - 범례는 반드시 그래프 바깥 위(`bbox_to_anchor`), 축 텍스트는 데이터 계열 색이 아닌 중립 회색(SUBTLE) — 자세한 이유와 실패 사례는 SKILL.md Step 4-2-5.
- **표**: `dataMatrixRow`/`dataMatrixCell` 기반. 헤더 행(ACCENT_LIGHT 배경) + 데이터 행(라벨 셀도 중앙정렬, v6부터). `type: "money"`는 금액 규모에 맞춰 억/백만/만원을 자동 선택(`fmtEok`), `type: "percent"`는 화살표+색상(양수 초록 `#1E8E5A` ▲ / 음수 빨강 `#C0392B` ▼).
- `rows`를 생략하면 표 없이 차트만 나온다(고용현황처럼 월별 컬럼이 너무 많아 표로 못 담는 경우).
- **성장률 계산 부호 규칙**: 기준값이 음수일 수 있는 지표(영업이익·순이익·자본 등)는 분모를 `abs(v0)`로 계산해야 부호가 직관과 맞는다. 자세한 배경은 SKILL.md 4-2-4.

## 6. 새 프로젝트에 적용하는 법

Claude에게 다음과 같이 요청하면 됩니다:

```
이 디자인 시스템 MD를 참고해서
[엑셀 파일명] 의 [시트명] 시트를 기반으로
기업별 1페이지 PDF/Word 파일을 만들어줘.

목차(섹션):
1. [섹션 이름1] — 엑셀 컬럼: [컬럼명]
2. [섹션 이름2] — 엑셀 컬럼: [컬럼명]
...

오버뷰 표 항목:
- [라벨]: [컬럼명]
- [라벨]: [컬럼명]
...
(10개 넘으면 2단 나란히 배치를 자동으로 고려할 것 — SKILL.md Step 3-1)
```

## 8. 주요 수치 참고표

| 변수명 | DXA 값 | 환산 |
|--------|--------|------|
| PAGE_WIDTH | 12240 | A4 너비 |
| PAGE_HEIGHT | 15840 | A4 높이 |
| MARGIN | 1080 | ≒ 1.9cm |
| CONTENT_WIDTH | 10080 | = PAGE_WIDTH - MARGIN×2 |
| 라벨 컬럼 폭 (단일 표) | 2400 | ≒ 4.2cm |
| 값 컬럼 폭 (단일 표) | 7680 | = CONTENT_WIDTH - 2400 |
| 2컬럼 간격 셀 | 400 | ≒ 0.7cm |
| 2단 오버뷰 그룹 간격(GAP) | 360 | ≒ 0.6cm |
| 2단 오버뷰 그룹당 너비 | 4860 | = (CONTENT_WIDTH - 360) / 2 |
| 2단 오버뷰 라벨 컬럼 폭 | 2100 | 고정값 |
| 2단 오버뷰 값 컬럼 폭 | 2760 | = 그룹너비 - 2100 |
| 단일행 기본 높이(margin) | 90 | `rowVMargin(1)` = `210×1-120` |
| span=2 행 margin | 300 | `rowVMargin(2)` = `210×2-120` |
| 필러 행 기준 빈 줄 높이 | 244 twips | `EMPTY_LINE_TWIPS` (실측 보정 계산용) |
| 성장 데이터 매트릭스 표 라벨 컬럼 | 2200 | `dataMatrixRow` 고정값 |
| 성장 데이터 표 compact 행 margin | 32~45 | 폰트도 함께 축소(라벨 15~16, 값 15~16) — 2페이지 초과 시 우선 조정 대상 |
| 차트 이미지 기본 폭 | 620~640px | `chartWidthPx`, 높이는 `chartAspect`로 자동 계산 |

> DXA → cm 환산: DXA ÷ 567 ≈ cm  
> docx 폰트 size → pt 환산: size ÷ 2 = pt
> twips = DXA (같은 단위, docx 라이브러리에서 혼용 표기)
